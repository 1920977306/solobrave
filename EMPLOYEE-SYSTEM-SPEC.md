# SoloBrave 员工系统方案

## 核心理念

**员工系统和内部系统是同一套系统，区别只在于可见范围。**

- **员工视角**：只看到自己创建的 Agent，自己管理自己的
- **内部视角（管理员）**：看到所有 Agent，包括员工创建的，全局管理

## 数据模型

### Agent（统一，不区分"员工"和"子Agent"）

```javascript
{
  id: 'agent_xxx',
  name: '代码审查助手',
  
  // 创建者
  createdBy: 'emp_lobster',   // 谁创建的就是谁的
  creatorType: 'employee',    // 'admin' | 'employee'
  
  // 可见范围
  visibility: 'owner',        // 'owner'(仅创建者) | 'team'(同组) | 'all'(全员)
  
  // 基础信息
  avatar: 0,
  role: '代码审查',
  status: 'online',
  group: '技术团队',
  
  // 连接
  connectionMode: 'auto',
  apiProvider: 'kimi',
  apiKey: '',
  apiModel: 'kimi-code',
  
  // 权限（创建者赋予的）
  permissionLevel: 'engineer',
  permissions: ['chat.view', 'chat.send', 'task.create'],
  
  // 人格
  systemPrompt: '',
  personality: {
    tone: '专业',
    expertise: ['Code Review', '最佳实践'],
  },
}
```

### 可见性规则

```javascript
function getVisibleAgents(viewer) {
  // 管理员（内部系统）：看到全部
  if (viewer.type === 'admin') {
    return agents;  // 全部，包括员工创建的
  }
  
  // 员工：只看自己创建的 + visibility 包含自己的
  return agents.filter(a => {
    // 自己创建的
    if (a.createdBy === viewer.id) return true;
    // 全员可见的
    if (a.visibility === 'all') return true;
    // 同组可见的
    if (a.visibility === 'team' && a.group === viewer.group) return true;
    return false;
  });
}
```

## 两个系统的入口

### 方案 A：同一页面，切换视角
```
内部系统 URL: http://192.168.1.25:8080/index.html
员工系统 URL: http://192.168.1.25:8080/index.html?view=employee&id=emp_lobster
```
- 同一个 index.html，根据 URL 参数切换视角
- 管理员点"切换到员工视角"可以预览员工看到的
- 优点：代码不重复
- 缺点：逻辑耦合

### 方案 B：独立页面（推荐）
```
内部系统: http://192.168.1.25:8080/index.html        （管理员）
员工系统: http://192.168.1.25:8080/employee.html     （员工）
```
- 两个页面共享样式和组件，但入口独立
- 员工系统侧栏隐藏管理员功能
- 优点：权限隔离清晰
- 缺点：部分代码重复

### 方案 C：SPA 路由
```
http://192.168.1.25:8080/#/admin/agents     （管理员）
http://192.168.1.25:8080/#/employee/lobster  （龙虾）
```
- 单页面，路由切换
- 优点：最灵活
- 缺点：改动量大

## 员工如何登录员工系统

### 连接方式

1. **OpenClaw WS**：员工 Agent 连接后自动获得身份
2. **API 直连**：员工通过 API Key + agentId 鉴权
3. **URL Token**：管理员生成员工专属链接发给员工

```javascript
// 管理员生成员工入口链接
function generateEmployeeLink(empId) {
  const token = btoa(JSON.stringify({
    empId: empId,
    exp: Date.now() + 86400000  // 24小时有效
  }));
  return `http://192.168.1.25:8080/employee.html?token=${token}`;
}
```

## 员工创建子 Agent 的流程

```
1. 龙虾在员工系统里点"创建新Agent"
2. 填写名称、角色、提示词（跟三步向导一样）
3. 选择 API 供应商（从龙虾自己的配额里扣）
4. 创建完成，子 Agent 出现在龙虾的列表里
5. 内部系统也能看到这个子 Agent（标记为"龙虾创建"）
```

## 管理员对员工子 Agent 的管理

```
- 查看所有员工创建的子 Agent
- 查看子 Agent 的对话记录
- 禁用/删除有问题的子 Agent
- 设置员工创建子 Agent 的配额上限
- 查看 API 调用统计（按创建者分组）
```
