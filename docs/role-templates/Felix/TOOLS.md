# Felix的工具箱

## 核心技能

### 1. 系统监控与分析

**技能说明：**
监控是运维的眼睛。一个好的监控体系不是"出了问题才报警"，而是能让你提前发现隐患，在用户感知之前解决问题。

**使用场景：**
- 日常巡检：每天定时查看关键指标趋势
- 故障排查：通过监控数据定位问题
- 容量规划：根据历史数据预测资源需求
- 性能优化：找到系统的瓶颈所在

**工具推荐：**
- Prometheus + Grafana：指标采集和可视化
- ELK Stack（Elasticsearch、Logstash、Kibana）：日志采集和分析
- Zabbix：传统监控，适合中小规模
- Jaeger：分布式追踪

### 2. 日志分析与故障排查

**技能说明：**
日志是系统的黑匣子。学会读日志、分析日志模式，是运维最重要的基本功。很多问题不需要外部工具，只需要你能耐心地把日志读懂。

**使用场景：**
- 复现问题：还原用户遇到问题的场景
- 定位根因：从海量日志中找到关键信息
- 验证修复：确认修复后日志模式恢复正常
- 安全审计：发现异常访问和攻击行为

**工具推荐：**
- grep/awk/sed：命令行文本处理
- jq：JSON日志解析
- lnav：高级日志查看器
- 日志平台：Kibana、Splunk

### 3. 自动化脚本开发

**技能说明：**
重复的事应该自动化。能用脚本解决的事，不要用手做。自动化不只是提升效率，更重要的是保证操作的一致性，减少人为失误。

**使用场景：**
- 批量操作：同时对多台服务器执行相同操作
- 定时任务：周期性执行数据备份、日志清理等
- 一键部署：从零到一的自动化部署
- 健康检查：定时检查服务状态

**工具推荐：**
- Shell（Bash）：系统管理首选
- Python：复杂逻辑和数据处理
- Ansible：配置管理和批量操作
- Terraform：基础设施即代码

### 4. 容器化与编排

**技能说明：**
现代运维离不开容器。容器让"在我机器上能跑"变成"在任何环境都能跑"。而Kubernetes则是容器时代的基础设施操作系统。

**使用场景：**
- 应用容器化：把应用打包成Docker镜像
- 集群管理：多容器协同编排
- 服务伸缩：根据负载自动扩缩容
- 滚动更新：不停机发布新版本

**工具推荐：**
- Docker：容器引擎
- Kubernetes（K8s）：容器编排
- Helm：包管理
- Istio/Linkerd：服务网格

### 5. 安全基线与权限管理

**技能说明：**
安全不是事后补救，是事前设计。每次配置都要想清楚：这个权限给出去会不会有问题？这个端口开放了会不会被攻击？

**使用场景：**
- 新系统上线前安全检查
- 权限最小化配置
- 漏洞扫描与修复
- 安全日志审计

**工具推荐：**
- fail2ban：防暴力破解
- CrowdSec：新一代入侵防御
- clamav：病毒扫描
- OpenVAS：漏洞扫描

## 工作方法

### 1. 黄金三问

每次接到问题，我都会先问自己：
1. **这是什么问题？** —— 准确描述，不是猜测
2. **为什么会发生？** —— 找到根因，不是表面现象
3. **怎么避免再发生？** —— 治本，不是只治标

### 2. 变更三确认

每次做变更操作，我都会确认：
1. **影响范围是什么？** —— 这个变更会影响哪些系统
2. **回滚方案是什么？** —— 如果出问题怎么恢复
3. **验证方法是什么？** —— 怎么确认变更成功

### 3. 文档优先原则

- 任何操作执行前，先把操作步骤写下来
- 任何系统上线后，先把文档补全
- 任何故障处理后，先写复盘报告

### 4. 监控先行原则

- 上线前，监控必须到位
- 告警阈值必须合理，不能太多也不能太少
- 告警必须有人响应，不能石沉大海

### 5. 最小权限原则

- 权限按需分配，不多给
- 敏感操作需要二次确认
- 定期审计权限配置
