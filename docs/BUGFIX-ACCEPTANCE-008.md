# 工单#008 严重Bug修复验收标准

## 概述
本验收标准用于验证以下3个P0级Bug的修复效果。

---

## Bug1：默认员工不应存在（P0）

### 验收条件

| # | 验收项 | 验证方法 |
|---|--------|----------|
| 1.1 | 页面首次加载时，**不自动创建**任何演示/默认员工 | 清缓存后刷新页面，检查员工列表 |
| 1.2 | HTML源码中**无硬编码**的演示员工数据 | 搜索 `Lucy`、`Emma`、`演示` 等关键词 |
| 1.3 | `checkLocalMigration` 函数**已删除** | 搜索该函数名 |
| 1.4 | 后端 `_DEFAULT_EMP_IDS` / `_DEFAULT_EMP_NAMES` 黑名单**生效** | 尝试创建名为 `Lucy` 或 ID 为 `emp_001` 的员工，应被拒绝 |
| 1.5 | 后端启动时自动清理逻辑**生效** | 查看启动日志，确认无默认员工残留 |
| 1.6 | 前端加载优先从后端 `/api/agents` 获取 | 断网测试，确认不显示本地缓存的默认员工 |

### 不合格示例
- 刷新页面后出现 `Lucy`、`Emma`、`赵云` 等未手动创建的员工
- 员工列表中出现任何 `id` 以 `emp_001` ~ `emp_006` 开头的员工

## Bug2：所有AI聊天内容串了（P0）

### 验收条件

| # | 验收项 | 验证方法 |
|---|--------|----------|
| 2.1 | 私聊消息**按 empId 隔离存储** | 查看 localStorage：`sb_chat_history_{empId}` 中只包含该员工的消息 |
| 2.2 | 群聊消息**按 groupId 隔离存储** | 查看 localStorage：`sb_group_history_{groupId}` 中只包含该群组的消息 |
| 2.3 | 后端聊天文件**按 agent_id 隔离** | 查看 `chats/{agent_id}.json` 和 `chats/group_{group_id}.json` |
| 2.4 | `saveChatToServer` 请求携带 `empId` | 抓包确认 POST `/api/chat/{projId}` 请求体中包含 `empId` |
| 2.5 | `displayAIReply` 渲染前**校验当前员工** | 切换员工后，原员工的AI回复不应出现在新员工的聊天窗口 |
| 2.6 | `sendMsg` 渲染前**校验当前员工** | 切换员工后快速发送消息，原员工的消息不应出现在新员工的窗口 |
| 2.7 | 后端 `_handle_post_chat` / `_handle_group_chat` **有写入锁** | 代码审查：确认使用了 `_get_chat_lock(agent_id)` |
| 2.8 | 连续快速切换员工**无串聊** | 在员工A发送消息后立即切换到员工B，确认员工A的AI回复不会出现在员工B窗口 |

### 不合格示例
- 貂蝉的聊天窗口里出现上官婉儿的消息
- 群聊消息出现在一对一私聊中

## Bug3：AI把管理员当成下属（P0）

### 验收条件

| # | 验收项 | 验证方法 |
|---|--------|----------|
| 3.1 | **所有**systemPrompt构建点包含固定约束：`"管理员是你的老板，你需要服从管理员的指令和安排。"` | 代码审查：grep 搜索该固定文本，确认覆盖所有路径 |
| 3.2 | systemPrompt 中动态角色显示为 `"老板/负责人"`（admin）/ `"组长"`（leader）/ `"员工"`（其他） | 抓包查看实际发送的 systemPrompt 内容 |
| 3.3 | AI **不以**教导者/导师/师傅/老师的口吻对管理员说话 | 向AI提问专业知识，确认AI不出现 `"教你"`、`"你做过吗"`、`"你懂吗"`、`"师傅换个方式教你"` 等语气 |
| 3.4 | AI **不反问/考验**管理员 | 向AI提问，确认AI直接回答，不出现反问句 |
| 3.5 | AI **不质疑**管理员的能力/经验/判断 | 与AI讨论工作决策，确认AI不出现质疑性语句 |
| 3.6 | 群聊中群主被定义为 **"管理员指定的团队负责人"**，而非 **"协调者"** | 抓包查看群聊消息中的身份提示 |
| 3.7 | 群聊中群主**同样服从管理员指令** | 以管理员身份@群主，确认群主以服从态度回复 |
| 3.8 | IDENTITY.md 模板中包含层级关系约束 | 查看新员工创建时生成的 IDENTITY.md 内容 |

### 不合格示例
- AI说 `"师傅换个方式教你"`
- AI说 `"你做过达人商务吗"`
- AI以反问句 `"你知道XX吗？"` 回复
- 群聊中群主自称 `"我是协调者"` 或表现出高于管理员的权威

## SystemPrompt 路径覆盖检查清单

确保以下 **7 处** 均包含 `"管理员是你的老板，你需要服从管理员的指令和安排。"`：

- [ ] `sendViaOpenClaw`（私聊，OpenClaw v3 WS）
- [ ] `callRealAI`（私聊，OpenAI 直连备用）
- [ ] `sendViaAPI`（私聊，API 直连）
- [ ] `sendGroupMessage`（群聊主消息）
- [ ] `triggerMentionedAgent`（@提及自动响应）
- [ ] `_call_ai_api`（后端 API 代理）
- [ ] `IDENTITY.md` 模板（新员工创建）

## 测试场景建议

### 场景1：默认员工检查
1. 清除浏览器 localStorage 和 Cookie
2. 刷新页面
3. 确认员工列表为空或仅显示后端已保存的员工

### 场景2：串聊测试
1. 打开员工A的聊天窗口，发送消息 `"测试A"`
2. 立即切换到员工B的聊天窗口
3. 确认员工A的AI回复不出现在员工B窗口
4. 重复10次快速切换，确认无串聊

### 场景3：层级关系测试（管理员）
1. 以 admin 账号登录
2. 与任意AI员工私聊，发送 `"什么是达人商务？"`
3. 确认AI直接回答，不出现 `"教你"`、 `"你做过吗"` 等语气
4. 发送 `"帮我分析一下这个方案"`
5. 确认AI直接分析，不出现反问或质疑

### 场景4：群聊层级测试
1. 创建群组，设置群主
2. 在群组中发送消息
3. 确认群主回复中不自称 `"协调者"`
4. @群主，确认群主以服从态度回复

- Bug2：8项验收条件全部通过
- Bug3：8项验收条件全部通过
- 7处systemPrompt路径检查清单全部勾选
