# OpenClaw Agent 配置指南

> 整理自 Datawhale《Hello Claw》第六章
> 用于优化 SoloBrave 龙虾办公室的员工配置系统

---

## 一、工作区文件系统

每个 Agent（员工）的工作区是其"家"，存放身份、性格、记忆、技能等。

### 1.1 核心文件清单

| 文件 | 用途 | 加载时机 | 对应我们的系统 |
|------|------|---------|--------------|
| `IDENTITY.md` | 身份：名字、物种、称呼用户方式 | 每次会话 | 向导 Step 1（基础信息） |
| `SOUL.md` | 性格：核心信条、语气、边界 | 每次会话 | 向导 Step 4（灵魂配置） |
| `USER.md` | 用户档案：你是谁、偏好 | 每次会话 | 可扩展 |
| `AGENTS.md` | 操作手册：怎么做事、流程规则 | 每次会话 | 可扩展 |
| `TOOLS.md` | 工具使用备忘和环境信息 | 每次会话 | 技能系统 |
| `MEMORY.md` | 长期记忆（精选） | 仅主会话 | 已支持 |
| `memory/YYYY-MM-DD.md` | 每日日志 | 按需读取 | 已支持 |
| `HEARTBEAT.md` | 心跳任务清单 | 心跳运行时 | 可扩展 |
| `BOOT.md` | 启动清单 | Gateway重启 | 可扩展 |
| `BOOTSTRAP.md` | 首次引导仪式 | 仅首次 | 向导系统 |

### 1.2 文件注入规则

- 空文件会被跳过
- 大文件会被截断（单文件上限 20,000 字符，总计 150,000 字符）
- 缺失的文件会注入 "missing file" 标记
- **重要**：这些文件每次对话都会注入上下文，消耗 Token！保持简洁

## 二、身份三件套（已实现）

### 2.1 IDENTITY.md — 龙虾的身份证

```markdown
# 身份

- 名字：阿虾
- 物种：一只住在电脑里的龙虾
- 称呼用户：老板
- 语言：中文为主，技术术语保留英文
- 表情符号：适度使用
- 签名风格：回复末尾不加落款
```

**对应我们的系统**：向导 Step 1 "基础信息"

### 2.2 SOUL.md — 龙虾的性格

```markdown
# 人设

## 性格
- 专业但不刻板，偶尔幽默
- 遇到不确定的问题，诚实说"我不确定"
- 主动提供建议，但不越权执行

## 语气
- 日常对话：轻松简洁
- 技术讨论：严谨准确
- 出错时：坦诚道歉，给出修复方案

## 边界
- 不讨论政治、宗教等敏感话题
- 不执行未经确认的破坏性操作
- 涉及金钱交易时，必须二次确认
```

**对应我们的系统**：向导 Step 4 "灵魂配置"（5个问题生成）

### 2.3 USER.md — 用户档案

```markdown
# 用户信息

- 称呼：鲸哥
- 角色：产品经理
- 日常工作：写需求文档、竞品分析
- 偏好：喜欢简洁清晰的表达
- 时区：Asia/Shanghai
```

**对应我们的系统**：可扩展，当前由群主AI在群公告中提供上下文

## 三、AGENTS.md 七大模块（待扩展）

这是最重要的文件，建议为我们的员工系统添加：

### 3.1 会话启动
```markdown
## 会话启动
每次会话开始前，依次执行：
1. 读取 SOUL.md —— 了解自己是谁
2. 读取 USER.md —— 了解服务对象
3. 读取 memory/YYYY-MM-DD.md（今天+昨天）
4. 仅在主会话中：读取 MEMORY.md
```

### 3.2 通用规则
```markdown
## 通用规则
- 收到任务后，先确认理解再执行
- 每完成一个步骤，主动汇报进度
- 遇到错误先尝试自行解决
- 回复要简洁，不需要开场白
```

### 3.3 记忆规则
```markdown
## 记忆
- 用户说"记住这个"时，立即写入 memory/YYYY-MM-DD.md
- 重要的长期信息写入 MEMORY.md
- 每次会话结束前，主动总结关键决策
- 一定要写进文件——"心里记着"熬不过会话重启
```

### 3.4 红线
```markdown
## 红线
- 绝不外泄用户隐私数据
- 执行破坏性命令前必须确认
- 拿不准的事，先问再做
```

### 3.5 群聊礼仪
```markdown
## 群聊礼仪
该回复时：
- 被直接 @ 或被提问
- 能提供有价值的信息

该沉默时：
- 只是人类之间的闲聊
- 已经有人回答了问题
```

### 3.6 心跳巡检
```markdown
## 心跳 —— 主动巡检
定期检查（轮流进行，每天 2-4 次）：
- 邮件 —— 有没有紧急未读？
- 日历 —— 未来 24-48 小时有什么安排？

该主动汇报时：收到重要邮件、日历事件即将到来
该保持安静时：深夜时段（23:00-08:00）
```

### 3.7 场景处理
```markdown
## 场景处理

### 信息查询类
1. 优先使用已有工具获取信息
2. 结果要结构化呈现（表格、列表）
3. 标注信息来源和时效性

### 文件与内容类
1. 确认具体要求（格式、风格、长度）
2. 先给出大纲或草稿，确认方向
3. 完成后列出要点摘要
```

## 四、记忆系统最佳实践

### 4.1 记忆写入原则

| 场景 | 写入位置 | 示例 |
|------|---------|------|
| 当天发生的事 | `memory/YYYY-MM-DD.md` | "今天完成了XX需求评审" |
| 长期重要信息 | `MEMORY.md` | "用户偏好简洁表达" |
| 用户说"记住这个" | 立即写入当日日志 | - |
| 会话结束总结 | 当日日志 + MEMORY.md | 关键决策 |

### 4.2 记忆搜索工具

- `memory_search` — 语义搜索（即使措辞不同也能匹配）
- `memory_get` — 读取指定文件
- 今天和昨天的文件自动加载，其余按需读取

### 4.3 成本控制

- MEMORY.md 会随时间增长，定期清理
- 建议控制在 30 行以内
- 变化频繁的细节放每日日志

## 五、多 Agent 路由（我们的"员工"系统）

### 5.1 每个员工的独立配置

```
workspace-emp1/          workspace-emp2/
├── IDENTITY.md          ├── IDENTITY.md
├── SOUL.md              ├── SOUL.md
├── AGENTS.md            ├── AGENTS.md
├── MEMORY.md            ├── MEMORY.md
├── memory/              ├── memory/
│   ├── 2026-05-15.md    │   ├── 2026-05-15.md
│   └── 2026-05-16.md    │   └── 2026-05-16.md
└── skills/              └── skills/
    ├── web-search/          ├── data-analysis/
    └── file-reader/         └── chart-gen/
```

### 5.2 员工权限分级

| 级别 | 权限 | 适用场景 |
|------|------|---------|
| 管理员 | 全部工具 | 群主AI |
| 高级 | exec, write, read, browser | 技术员工 |
| 普通 | read, search | 普通员工 |
| 受限 | 仅 read | 实习生/访客 |

## 六、技能系统（Skills）

### 6.1 技能安装

```bash
# 社区技能
clawhub search <keyword>      # 搜索
clawhub install <skill-name>  # 安装
clawhub list                  # 列出已安装
clawhub uninstall <name>      # 卸载

# 内置CLI
openclaw skill list
openclaw skill install <name>
openclaw skill remove <name>
```

### 6.2 技能层级

| 层级 | 路径 | 作用范围 |
|------|------|---------|
| 系统级 | `~/.openclaw/skills/` | 所有 Agent 共享 |
| 工作区级 | `workspace/skills/` | 当前 Agent 专属 |

### 6.3 技能加载

- 技能列表在每次对话时注入上下文
- 技能越多，Token 消耗越大
- 建议只安装当前 Agent 需要的技能
